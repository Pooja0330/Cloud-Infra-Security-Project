# Step 3: Enable Monitoring (CloudTrail & GuardDuty)

## Purpose
Track API activity and detect threats in your AWS account.

## How to Use
- Run `cloudtrail_setup.sh` to enable CloudTrail logging.
- Run `guardduty_setup.sh` to enable GuardDuty monitoring.
- Check AWS Management Console for logs and findings.

## Notes
- Ensure S3 bucket permissions are correct for CloudTrail.
