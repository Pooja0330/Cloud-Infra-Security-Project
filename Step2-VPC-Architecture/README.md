# Step 2: VPC Architecture Setup

## Purpose
Create Virtual Private Cloud with public/private subnets, route tables, security groups, and NACLs.

## How to Use
- Run `vpc_setup_commands.sh` script to create resources.
- Review `security_groups_and_nacls.md` for SG and NACL rules details.
- Verify VPC, subnets and security configurations in AWS Console.
